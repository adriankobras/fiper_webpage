Failure Prediction at Runtime without Failure Data for Generative Robot Policies

NeurIPS 2025
Ralf Römer*,1, Adrian Kobras*,1, Luca Worbis1, Angela P. Schoellig1
1 Learning Systems and Robotics Lab, Technical University of Munich
*Indicates Equal Contribution
Paper Code arXiv

Abstract

We introduce FIPER (Failure Prediction at Runtime), a lightweight, model-agnostic framework that predicts failures of generative imitation-learning (IL) robot policies early and without using failure data. FIPER combines (i) Random Network Distillation in the policy’s observation embedding (RND-OE) with (ii) Action-Chunk Entropy (ACE) computed from batched policy samples. Both scores are statistically calibrated via conformal prediction on a small set of successful rollouts and aggregated over short time windows. A failure alarm is raised iff both calibrated scores exceed their time-varying thresholds. We evaluate our framework in five simulation and real-world environments where various types of failures can occur. Our results show that FIPER better distinguishes true failures from OOD situations and predicts failures earlier and more accurately than existing methods. We thus consider FIPER an important step towards more interpretable and safer generative robot policies.

Motivation

Recent advances in generative imitation learning (IL) robot policies have enabled robots to solve increasingly complex tasks. However, safe deployment remains challenging:

  • Unpredictable policy failures can arise from distribution shifts or compounding errors.
  • Existing methods either:
    • Flag any OOD observation/action (high false alarms)
    • Use VLMs to retroactively detect failure (too late)
    • Rely on existing failure data (often unsafe/impractical to collect)
    • Use policy inputs only or outputs only, missing crucial cues
    • Fail to account for the multi-modal action capabilities of generative policies
We address these challenges with FIPER, a framework for failure prediction at runtime that provides early and accurate warnings without requiring failure examples.

Figure 1: Generative models are able to learn multi-modal action distributions. In contrast to other methods, FIPER is specifically designed to handle these multi-modal action capabilities.

Method

FIPER predicts policy failures during runtime by combining two complementary indicators:

  • Observation-Based Score (RND-OE): Random Network Distillation applied in the policy’s own observation embedding space detects when current observations deviate from those seen in successful rollouts. Applying RND to the observation embeddings instead of raw observations improves robustness to irrelevant distribution shifts.
  • Action Chunk Entropy (ACE): Batched action sequences are evaluated via a novel entropy score in end-effector space, capturing persistent ambiguity in the robot’s intended behavior. Our proposed score is both computationally lightweight and effective at handling multi-modal action distributions.
Both scores are aggregated over short moving windows and calibrated with conformal prediction using only a small number of successful trajectories. A failure is flagged only when both indicators exceed their individual thresholds, yielding early and robust prediction while avoiding false alarms on benign out-of-distribution states.

Evaluation

We evaluate FIPER across five diverse tasks using both diffusion- and flow-based policies. Calibration relies only on a small set of successful rollouts (50 in simulation, 10 in real-world). Across all environments, FIPER consistently predicts failures earlier and more accurately than observation-only or action-only baselines. By combining calibrated embedding-space novelty with action-space entropy, it offers a practical mechanism for safer deployment of generative robot policies.

Highlights

  • FIPER predicts failures of generative robot policies at runtime without requiring failure data.
  • Combines observation-based and action-based indicators for fast and robust failure prediction.
  • Uses conformal prediction to provide statistical guarantees on prediction performance.
  • Calibrated using only a small number of successful rollouts, enabling deployment without access to policy training data.
  • Outperforms existing methods in distinguishing true failures from out-of-distribution situations.
